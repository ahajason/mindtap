## 1. 原始需求（用户首条消息）

> 我现在感觉我原本的期望是实现移动端APP的，但现在移动端APP的优先级没那么高，但是现在不论是MAC还是Windows，他实现的PC端，总感觉想一个应用程序的后台，而并不是一个轻量的APP。我现在想修改节奏，优先做一小界面浮动的窗口快速记录。
>
> **核心痛点**：多维度生活记录操作繁琐，导致灵感瞬间流失、记录难以持续
>
> **目标用户**：点子爆发但极易遗忘、难以长期坚持记录的创意人群（如 ENFP）
>
> **唯一设计准则**：3 秒记录、1 秒查看、0 思考成本
>
> **基础特性**：无登录注册、全量数据本地存储、重记录轻查看、无冗余功能
>
> **TODO 缺陷**：V1.2 只有完成和未开始，不能完整满足核心功能需求。作为用户，需要在工作时通过待办轻松得记录当前的任务，避免后续遗忘。同时工作时，容易遗忘正在进行的任务，会经常忘了去更新。所以更需要一个**常驻的可以计时的浮动极简窗口**来提示。
>
> **架构约束**：同时满足现阶段的 MAC 和 Windows 的浮动窗口的实现架构

## 5. 4 状态 TODO 状态机

```
              ┌─────────┐
       ┌─────→│ 未开始  │←──── 创建时默认
       │      └────┬────┘
       │           │ 点击"开始计时"（保存并启动）
       │           ▼
       │      ┌─────────┐
       │      │ 进行中  │ ← FOCUS 标签 + 呼吸灯
       │      └────┬────┘
       │           │
       │     ┌─────┴─────┐
       │     ▼           ▼
       │  ┌──────┐    ┌──────┐
       │  │ 暂停 │    │ 完成 │  ← 完成 → 从未开始+已暂停清单消失
       │  └──┬───┘    └──────┘
       │     │ 点 ⏸ 继续
       │     ▼
       │  进行中（继续累加时间）
       │
       └─────────── 任何状态可"重新激活"为进行中
```

| 状态 | 视觉 | 在浮动窗逻辑 |
|------|------|-------------|
| 未开始 | 灰标签 | 出现在"切换"下拉清单中 |
| 进行中 | 绿标签 FOCUS + 呼吸灯 | 出现在主 Focus 块 |
| 暂停 | 橙标签 | 出现在"切换"下拉清单中，带已计时数 |
| 完成 | 绿底按钮 | 移到历史 |

**关键命名**：
- 暂停 ≠ 完成
- 暂停 = 临时停止（保留累计时间）
- 完成 = 永久归档（从待办清单消失）

**视觉权重（高→低）**：
1. `⏵ 开始计时`（蓝实心，CTA）
2. `⏸ 暂停`（浅蓝底，文字按钮）
3. `✓ 完成`（淡绿底，文字按钮）
4. `📅 稍后/明天`（灰底，次要）
5. 类型/切换 `⇄`（灰底，最次要）

## 6. 关键语义决策

### 6.1 "保存" = "开始计时"
- 用户在 v6 提议：保存按钮直接作为计时起点
- 0 思考成本：用户输入完 → 点 → 自动 Focus + 启动计时
- 表单 `开始计时` 按钮是 V1.2 `保存` 按钮的进化版
- 不再有"保存到待办但不计时"的中间态

### 6.2 "切换" ≠ "新建"
- 切换：从已有的"未开始+已暂停"清单中选一个成为 Focus
- 新建：创建新任务
- 切换按钮在窗体右上角（顶靠右下拉）

### 6.3 Focus ≠ TODO
- 进行中的任务**视觉上不是 TODO**（绿标签 + 呼吸灯 vs 蓝标签）
- 命名上不混用
- 但底层数据是同一张表（统一表 + JSON 扩展字段）

### 6.4 暂停 < 完成（视觉权重）
- 暂停是临时状态，可继续（重要）
- 完成是终态，消失（次要）
- 因此暂停按钮视觉更突出（浅蓝）> 完成（淡绿）

## 7. 视觉规范（Liquid Glass）

### 7.1 颜色系统
| 用途 | 颜色 | 用例 |
|------|------|------|
| 主蓝（CTA / Focus 进行中时间） | `#007AFF` | 开始计时按钮、暂停文字、时间数字 |
| 主绿（成功 / 进行中标识） | `#34c759` | FOCUS 标签、暂停图标、已完成 |
| 绿深（标签字色） | `#1f8a3f` | FOCUS 文字、已完成文字 |
| 橙（暂停） | `#ff9500` | 暂停清单项标签 |
| 橙深 | `#cc7a00` | 暂停标签字色 |
| 灰（次要） | `#86868b` | 标签字色、未开始标签 |
| 文字主色 | `#1d1d1f` | 标题、内容 |

### 7.2 透明度梯度
| 层级 | rgba | 用途 |
|------|------|------|
| 窗体主底 | `rgba(255,255,255,0.42)` | 浮动窗主容器 |
| 折叠态底 | `rgba(255,255,255,0.34)` | 折叠态（更轻） |
| 卡片元素 | `rgba(255,255,255,0.5)` | 内部按钮 |
| 输入框 | `rgba(255,255,255,0.45)` | textarea |
| 高亮 | `rgba(0,122,255,0.12)` | 蓝底按钮 |
| 高亮 | `rgba(52,199,89,0.12)` | 绿底按钮 |
| 标签 | `rgba(52,199,89,0.18)` | FOCUS 标签底 |
| 下拉 | `rgba(255,255,255,0.7-0.85)` | 下拉面板 |

### 7.3 模糊
- `backdrop-filter: blur(30px) saturate(180%)`（折叠态）
- `backdrop-filter: blur(32px) saturate(200%)`（展开态）

### 7.4 边框
- 1px `rgba(255,255,255,0.6-0.65)` 内边框
- `box-shadow: 0 1px 0 rgba(255,255,255,0.7) inset, 0 8-10px 24-28px rgba(0,0,0,0.08-0.1)` 立体感

### 7.5 圆角
- 折叠态：`18px`（胶囊）
- 展开态主容器：`18px`
- 内部按钮：`5-8px`（小按钮 5-6，按钮组 7-8）

### 7.6 动画
- 呼吸灯：`pulse 1.6s ease-in-out infinite`（Focus 状态指示）
- 折叠/展开：需决定 200-300ms ease-out 缩放 + 透明度（待节 5 详细）
- 状态切换：标签颜色淡入淡出 200ms

## 8. 跨平台浮动窗架构（Tauri 2）

### 8.1 多窗口
```
Tauri App
├── MainWindow (主窗)
│   ├── 设置
│   ├── 统计
│   └── 数据管理
└── FloatingWindow (浮动窗)
    ├── 折叠态 (320×36)
    ├── 展开态 (360×280)
    └── 始终置顶 (alwaysOnTop: true)
```

### 8.2 关键 Tauri 配置
- `transparent: true`
- `decorations: false`（无标题栏）
- `alwaysOnTop: true`
- `resizable: false`
- `skipTaskbar: true`（macOS 26 dock 隐藏）
- `focus: false`（点击不抢焦）
- `shadow: false`（用 CSS 自绘）

### 8.3 跨平台差异
- **macOS**：NSPanel 通过 `panel` 模式（不抢焦 + 浮窗）+ `setFloatingPanel` 级别
- **Windows**：WS_EX_TOOLWINDOW | WS_EX_LAYERED + topmost flag（Tauri 2 内置支持）

### 8.4 数据流
- 浮动窗与主窗共享 SQLite（同进程）
- 浮动窗事件 → Tauri event bus → 主窗自动响应
- 全局快捷键（macOS Cmd+Shift+Space，Windows Ctrl+Shift+Space）→ 显示/隐藏浮动窗

### 8.5 状态持久化
- 浮动窗位置：`localStorage` 或 Tauri Store
- 折叠/展开：内存
- 启动时自动启动浮动窗

## 9. 核心用户旅程

### 9.1 3 秒记录
1. `Cmd+Shift+Space` → 浮动窗显示（折叠→展开）
2. 输入文字 `回复张三关于 Q3 的邮件`（< 1s）
3. 点 `⏵ 开始计时`（< 0.5s）
4. 浮动窗折叠 → Focus 块显示该任务 + 计时
5. **总耗时 < 3s**

### 9.2 1 秒查看
- 浮动窗常驻桌面任意位置
- 一眼看到：当前 Focus + 已计时数
- 无需打开主窗

### 9.3 切换 Focus
1. 浮动窗展开
2. 点 `⇄ ▼` 顶靠右下拉
3. 下拉显示"未开始+已暂停"清单
4. 点目标项右侧 `⏵` 切换
5. 浮动窗自动回到主 Focus 块 + 重新计时

### 9.4 暂停 vs 完成
- `⏸ 暂停` → 状态→暂停，时间累计保留，出现在切换下拉
- `✓ 完成` → 状态→完成，从未开始+已暂停清单消失，归档

## 12. v15 修订（基于用户 3 个质疑）

### 12.1 D1 · 3 表拆分（待用户选 A/B/C）
- **质疑**：todo/idea/check_in 用同一 record 表不合理
- **回应**：
  - todo（待办 4 状态） / idea（灵感 1 状态） / check_in（频次）
  - 3 者语义/查询模式/UI 完全不同
  - 推荐 **A 3 独立表**（task / idea / check_in）
  - 备选 B 单表+kind（迁移简单但 nullable 多）/ C 2 表（折中）
- **Schema** 见 v15

### 12.2 D2 · 首次 focus 时间字段（✅ 已加）
- 缺 `first_focused_at`（首次激活起点）— 用于"任务总跨度"
- 缺 `focused_count`（累计激活次数）— 用于"重启了 N 次"洞察
- V14 payload → V15 task 表字段
- **4 个时间字段语义区分**：
  - `first_focused_at` 首次激活（不变）
  - `focus_started_at` 本次激活（暂停清零）
  - `paused_at` 本次暂停起点
  - `duration_ms` 累计计时
  - `focused_count` 累计激活次数

### 12.3 D3 · 完成态在 Focus 块保留（待用户确认视觉）
- **质疑**：完成 → 立即归档 → Focus 块清空
- **修正**：完成 → Focus 块变体（done 态）→ 新任务覆盖时归档
- **3 态 Focus 块视觉**：
  - `active`：呼吸灯 + FOCUS 绿标 + 蓝时间 + 绿 ⏸
  - `done`：静默圆点 50% + DONE 淡绿 + 灰冻结时间 + 蓝 ↶ 撤销
  - `empty`：灰圆点 + EMPTY 灰标 + 提示文字 + 蓝 + 新建
- **关键流程**：
  - active → done：归档 duration_ms，Focus 块变 done
  - done → active（撤销）：focus_started_at = now，duration 保留
  - 新任务 ⏵ / ⇄ 切换：旧 done 自动 archived_at = now

## 13. v16 D4 · 3 表 + 1 通用 record 表（用户补充）

### 13.1 需求
- 3 源表独立（task / idea / check_in）已锁定
- 补充：需要一个**通用 record 表**通过"事件"形式关联创建
- 用途：主窗"全部记录"时间线**统一列表展示**

### 13.2 3 方案对比（用户选 B）
- ~~A 触发器自动同步~~ — 否决
- **B 业务层事务双写** ✅ 锁定
- ~~C 纯视图 UNION~~ — 否决

### 13.3 B 方案核心（事务包裹）
- 源表 INSERT + record INSERT 同事务，全部成功/回滚
- `tx.last_insert_rowid()` 取 source_id
- UPDATE 同步 record.content/status
- DELETE 级联 record
- 业务层不直接 SQL，必须走 create_*/update_*/delete_*

### 13.4 3 关注点推荐答案（v17）
- **① 迁移**：检测旧表 → 建新表 → 拆 3 源 → 回填 record → 校验 + 备份 30 天
- **② 筛选**：顶部 4 tab（全部/任务/灵感/打卡）+ 时间维度 + ⌘F 搜索，不做多选
- **③ 归档**：推荐 A 隐藏归档（archived_at 非空 → 主窗不显示，归档 tab 可见）

### 13.5 主窗列表 SQL
- 全部：`SELECT * FROM record ORDER BY created_at DESC`
- 隐藏归档：`WHERE kind != 'task' OR source_id NOT IN (SELECT id FROM task WHERE archived_at IS NOT NULL)`
- 归档 tab：JOIN task 取 archived_at
- 点击列表项：kind + source_id 跳转源表详情
