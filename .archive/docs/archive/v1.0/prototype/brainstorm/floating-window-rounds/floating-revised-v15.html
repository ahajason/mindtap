D1 数据模型 · 3 表拆分

原 V1.2 用单表 record + type 字段存 todo/idea/check_in。V1.3 引入 4 状态机后，todo 需要专属字段（status / duration / focus 时间），idea 永远只有"已记录"一种状态，check_in 是频次维度。语义/查询模式/UI 都不同，应拆 3 表。

3 表方案对比

A · 3 独立表

task（待办 4 状态）
idea（灵感）
check_in（打卡）

B · 单表 + kind

record + kind: 'task'|'idea'|'check_in' + 状态字段 nullable

C · 2 表拆分

record（task+idea 合并）
check_in（独立）

推荐 A · 3 独立表 Schema

-- 待办（V1.3 核心，4 状态机）
CREATE TABLE task (
  id                  INTEGER PRIMARY KEY AUTOINCREMENT,
  content             TEXT    NOT NULL,
  status              TEXT    NOT NULL DEFAULT 'pending',
  -- 'pending' | 'active' | 'paused' | 'done'
  duration_ms         INTEGER NOT NULL DEFAULT 0,   -- 累计计时（暂停/完成时累加）
  first_focused_at    INTEGER,                       -- 首次激活时间（不变，用于"任务总跨度"）
  focus_started_at    INTEGER,                       -- 本次激活起点（暂停清零）
  paused_at           INTEGER,                       -- 本次暂停起点
  focused_count       INTEGER NOT NULL DEFAULT 0,    -- 累计激活次数
  due_at              INTEGER,                       -- 截止时间（📅稍后/明天）
  created_at          INTEGER NOT NULL,
  updated_at          INTEGER NOT NULL,
  archived_at         INTEGER                        -- 归档时间（被新任务覆盖时填）
);
CREATE INDEX idx_task_status ON task(status);
CREATE INDEX idx_task_due_at ON task(due_at);

-- 灵感（极简）
CREATE TABLE idea (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  content     TEXT    NOT NULL,
  created_at  INTEGER NOT NULL
);
CREATE INDEX idx_idea_created_at ON idea(created_at);

-- 打卡（频次统计）
CREATE TABLE check_in (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  habit       TEXT    NOT NULL,         -- 习惯名（如"早起""跑步""冥想"）
  note        TEXT,                     -- 可选备注
  checked_at  INTEGER NOT NULL
);
CREATE INDEX idx_check_in_habit     ON check_in(habit);
CREATE INDEX idx_check_in_checked_at ON check_in(checked_at);

推荐 A：V1.2 → V1.3 已有数据迁移成本，但 task 的 4 状态机/计时字段不应让 idea/check_in 承担。A 长期更清晰。

D2 payload 扩展 · 首次 focus 时间 + 激活次数

V14 payload 缺 first_focused_at（首次激活起点，用于"任务总跨度"）。总跨度 ≠ 累计计时：跨度 = first_focused_at → 现在（含暂停）；计时 = 累加 active 片段。

4 个时间字段的语义

字段语义何时变
first_focused_at首次激活起点pending → active 时设；后续不变
focus_started_at本次激活起点每次 active 重新设；暂停时清
paused_at本次暂停起点active → paused 时设
duration_ms累计计时暂停/完成时累加（now - focus_started_at）
focused_count累计激活次数每次 pending/paused → active 时 +1

统计指标对照

用户问题用哪个字段
"这个任务做了多久？"duration_ms
"这个任务跨度多少天？"now - first_focused_at
"中间暂停了多久？"跨度 - 计时
"这个任务重启过几次？"focused_count - 1
"今天累计做了多少？"按日聚合 duration_ms

D3 完成态保留在 Focus 块 + 撤销继续

原 V14：完成 → 立即归档 → Focus 块清空。问题：刚完成的事看不到，撤销需查历史。
新设计：完成 → Focus 块变体（done 态视觉）→ 仍占位 → 新任务 ⏵ 开始计时时自动归档旧 done → 允许"撤销继续"（done → active）。

Focus 块 3 种态视觉对比

DONE 写 Q3 计划 00:32:14 ↶
态 3 · empty 无任务
EMPTY 点击记录第一个任务 +
