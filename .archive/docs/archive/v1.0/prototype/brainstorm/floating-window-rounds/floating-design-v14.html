2.3 SQLite Schema（V1.2 单表扩展）

CREATE TABLE record (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  type         TEXT    NOT NULL,          -- 'todo' | 'idea' | 'check_in'
  status       TEXT    NOT NULL DEFAULT 'pending',  -- 'pending' | 'active' | 'paused' | 'done'
  content      TEXT    NOT NULL,
  payload      TEXT    NOT NULL DEFAULT '{}',  -- JSON: { duration_ms, paused_at, focus_started_at, ... }
  created_at   INTEGER NOT NULL,
  updated_at   INTEGER NOT NULL
);

-- 索引
CREATE INDEX idx_record_status ON record(status);
CREATE INDEX idx_record_type   ON record(type);

2.4 payload 字段（计时用）

字段类型说明
duration_msnumber累计计时（毫秒），暂停时冻结
focus_started_atnumber | null本次"开始计时"时间戳；暂停时保留，重新激活时叠加
paused_atnumber | null本次暂停时间戳
due_atnumber | null截止时间（📅稍后/明天快捷）

2.5 状态机不变量（写操作必须满足）

  • L1：同时 只能有一个 status='active' 的 record（全局唯一 Focus）
  • L2：从 paused → active 时，focus_started_at = now()，duration_ms 保持累加
  • L3：从 active → paused 时，duration_ms += now() - focus_started_at，focus_started_at = null，paused_at = now()
  • L4：从 active → done 时，累计完 duration_ms 后归档
  • L5：从 done → * 不允许（终态）。可"重新激活"为 active（重置 duration_ms）

3.3 关键 Tauri 命令（Rust ↔ JS）

命令方向用途
record_start(content, type, due_at?)JS → Rust新建 + 启动计时，原子事务
record_pause(id)JS → Rust暂停，累加 duration_ms
record_resume(id)JS → Rust从暂停恢复，update focus_started_at
record_complete(id)JS → Rust完成，归档
record_switch(id)JS → Rust切换 Focus：当前 active → pending，新 id → active
floating_show / hide / toggleJS → Rust控制浮动窗可见
evt_focus_changedRust → JSFocus 切换事件，浮动窗自动刷新
evt_tickRust → JS每秒推送（前端不计时）

4 核心交互流程

4.1 3 秒记录 · 端到端

  T+0.0s   按 Cmd+Shift+Space
  T+0.1s   浮动窗展开（折叠态 36px → 展开态 280px，200ms 动画）
  T+0.5s   用户输入文字 "回复张三关于 Q3 的邮件"
  T+1.5s   点 ⏵ 开始计时
  T+1.6s   写 DB（status='active', focus_started_at=now）
           浮动窗折叠（280px → 36px，200ms 动画）
  T+1.8s   主 Focus 块显示该任务 + 时间从 00:00:00 开始走
  T+2.0s   用户可继续工作

  ✓ 总耗时 < 3s

4.2 1 秒查看 · 任意时刻

  浮动窗常驻桌面任意位置
  一眼：● FOCUS 写 Q3 计划     00:12:34  ⏸
       ↑ 呼吸灯   ↑ 标签   ↑ 标题      ↑ 计时  ↑ 状态

  · 无需打开主窗
  · 折叠态 36px 可拖动
  · 鼠标 hover → 200ms 渐变高亮（可选）

4.3 切换 Focus

  1. 浮动窗展开
  2. 点 ⇄ ▼ 顶靠右下拉（顶靠右浮在主窗内）
  3. 下拉显示：未开始（灰）+ 已暂停（橙）清单
  4. 点目标项右侧 ⏵
  5. 后端原子操作：
     - 当前 active → pending (or paused)
     - 新 id → active (focus_started_at = now)
  6. 浮动窗主 Focus 块自动更新（evt_focus_changed）
  7. 下拉自动关闭，浮动窗回到默认展开态

4.4 暂停 vs 完成

⏸ 暂停

  active → paused
  · duration_ms += (now - focus_started_at)
  · 出现在切换下拉，带已计时
  · 状态机：可继续（paused → active）
  · 视觉：暂停按钮浅蓝（视觉更突出）

✓ 完成

  active → done
  · duration_ms 累加完毕
  · 从切换下拉消失
  · 归档到历史（主窗可见）
  · 视觉：完成按钮淡绿（视觉更弱）

4.5 截止快捷 📅 稍后 / 明天

  • 📅 稍后 → due_at = end_of_today(今天 23:59)
  • 📅 明天 → due_at = end_of_tomorrow(明天 23:59)
  • 仅设置 due_at，不影响 status（仍要点 ⏵ 开始计时）
  • 主窗列表按 due_at 排序显示

4.6 关闭主窗时浮动窗

  • macOS：主窗关 → 浮动窗保留（用户不退出 app，符合 Mac 习惯）
  • Windows：托盘图标常驻，右键托盘 → 退出
  • 自启动：浮动窗跟随系统启动后自动显示（折叠态）
