# 作息打卡 · 业务规格

> 用户在浮窗以双 time-picker 记录每日起止时间,主窗历史以时间线展示并支持日历视图。

## §1 功能需求（F）

- **F-1** 用户在浮窗快速打卡入口选择"作息",设置就寝时间与起床时间后提交一条作息记录
  - R-1.1 用户可通过两个时间选择器分别设置就寝时间与起床时间
  - R-1.2 就寝时间与起床时间均为必填字段,缺一不可提交
  - R-1.3 系统按就寝与起床时间自动计算睡眠时长
- **F-2** 用户在主窗历史列表中按时间倒序查看所有作息记录
  - R-2.1 列表按提交时间倒序排列
  - R-2.2 单条卡片显示就寝时间、起床时间、自动计算的睡眠时长
- **F-3** 用户可在主窗查看作息的日历视图
  - R-3.1 日历按天聚合,每格显示当日睡眠时长
  - R-3.2 无打卡日期不显示格子背景色
- **F-4** 用户可编辑或删除任意一条作息记录
  - R-4.1 编辑后就寝/起床时间更新,睡眠时长重新计算
  - R-4.2 删除后该记录从全部记录列表与日历视图中消失,操作不可恢复
- **F-5** 用户可设置工作日 / 周末不同的作息表
  - R-5.1 工作日表与周末表独立配置
  - R-5.2 当日应用哪张表按日历自动判断(周一-周五用工作日,周六周日用周末)
  - R-5.3 用户可关闭周末表(全部走工作日)
- **F-6** 用户可标记某日为"补眠日",补眠日不计入作息规律统计的连续性
  - R-6.1 补眠日需提前或当日标记
  - R-6.2 补眠日不计入连续天数计算

## §2 业务规则（R）

（无）

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供轻量级作息自我记录,服务用户的睡眠规律观察需求
- **G-2** 与本主题其他子场景共享同一基础记录结构与浮窗入口

### NonGoal

- **NG-1** 本文件不涉及"自动睡眠监测(穿戴设备)"——留 v2
- **NG-2** 本文件不涉及"睡眠分期分析(深睡/浅睡/REM)"——留 v2
- **NG-3** 本文件不涉及"Apple Health 集成"——留 v2
- **NG-4** 本文件不涉及"打卡通用数据模型与浮窗入口设计"——本主题所有子场景共享同一基础结构,具体字段约束由各场景自行规定——见 `_cross/foundation.md`
- **NG-5** 本文件不涉及"跨午夜睡眠归属"——按设备本地时区的"自然日"判定:同一天内提交算当天(0:30 入睡,8:00 起床 算前一天)——见 `_cross/timezone.md`
- **NG-6** 本文件不涉及"DST 切换日睡眠时长异常"——按时戳记,渲染层按 23h/25h 显示——见 `_cross/timezone.md`
- **NG-7** 本文件不涉及"自动睡眠监测(穿戴设备)"——留 v2
