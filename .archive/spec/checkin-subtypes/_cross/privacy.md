# 跨子场景隐私策略 · 业务规格

> 7 个子场景中敏感数据(情绪 / 复盘 / 服药)的统一隐私处理策略。

## §1 功能需求（F）

- **F-1** 用户可在主窗设置启用"私密模式"开关
  - R-1.1 开关默认关闭,开启后影响所有敏感子场景
  - R-1.2 开启后浮窗不显示情绪 / 复盘 note 文本预览,仅显示 emoji 或模板名
  - R-1.3 开启后 OS 通知不展示 note 内容
- **F-2** 用户可启用"启动锁"功能,打开 app 时需输入密码
  - R-2.1 启动锁默认关闭
  - R-2.2 启用后 macOS 启动应用 / 重新进入前台时验证密码
  - R-2.3 错误密码 3 次后等待 1 分钟
- **F-3** 数据导出时,用户可勾选"是否包含 note 文本"
  - R-3.1 默认包含,敏感子场景提示"含敏感内容"

## §2 业务规则（R）

- **R-0.1** 敏感子场景 = 情绪 / 复盘 / 服药;非敏感 = 习惯 / 饮水 / 作息 / 饮食
- **R-0.2** 数据库文件权限为 600(仅当前用户可读)
- **R-0.3** 删除操作不可恢复,无"软删除 30 天后清理"机制
- **R-0.4** settings.json 与 mindtap.db 不上传任何云端,备份由用户自行处理

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供"轻隐私"基线:默认无锁,但提供启用锁的开关;用户自决
- **G-2** 敏感数据在共享电脑场景下有兜底保护(浮窗 note 隐藏 + 启动锁可选)

### NonGoal

- **NG-1** 本文件不涉及"单条记录加密"——v1 数据库明文存储,留 v2
- **NG-2** 本文件不涉及"远程擦除 / 设备丢失保护"——本地单机产品,无云
- **NG-3** 本文件不涉及"7 个子场景各自的具体功能"——见 `../emotion.md` 等
