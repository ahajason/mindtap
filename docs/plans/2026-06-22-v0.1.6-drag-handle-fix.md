# V0.1.6 — Drag Handle 模式 (收口 V0.1.4 drag region 副作用)

> 创建: 2026-06-22
> 状态: 待用户审

## Why

V0.1.4 commit `20e9ac9` 把窗口拖动从 `NSWindow.setMovableByWindowBackground(true)` 切到 web 侧 `data-tauri-drag-region`,目的是修 V0.1.1 retro 暴露的 3 症状(文字选不中 / resize 失效 / cursor 不变)。

**切方案时漏了 Layer 3 反向副作用枚举**:`data-tauri-drag-region` 底层是 webkit CSS `-webkit-app-region: drag`,**会被 button / input / a 自动 reset 成 no-drag**。当前 `<div data-tauri-drag-region>` 包住整个窗口,主内容里全是 button / nav → 用户视觉上的"空白处"实际命中 button → drag 失效。

修法 = 缩小 drag region 到**专用拖动条**,其他区域不标 drag,自然恢复交互。

## What

拆整窗 drag region 到两个专用 drag handle:

1. **Sidebar 顶部 header 区**(左侧 drag handle)
2. **Main 顶部固定条**(右侧 drag handle,横跨 main 容器外缘)

外层 div 移除 drag region。其他区域(Sidebar nav、main 内容、main 滚动条)不加 drag region,默认 no-drag。

## 设计细节

### Sidebar header drag handle

- `<header data-tauri-drag-region>` 包住 h1 + 上下 padding
- 整个 header 区域(包括 padding)都可拖
- 不影响 nav(默认 no-drag,button 可点)
- macOS native traffic light 在 mt-10 上方,与 header 不重叠

### Main 顶部 drag handle

```tsx
<main className="flex-1 mr-3 mt-3 mb-3 overflow-y-auto p-[var(--spacing-6)]">
  {/* 拖动条:高度对齐 macOS titlebar 视觉(~36px),横跨 main 上方 */}
  <div data-tauri-drag-region className="h-9 -mx-[var(--spacing-6)] -mt-[var(--spacing-6)] mb-3" />
  <Outlet />
</main>
```

- `-mx` / `-mt` 让 drag handle 横跨 main 容器的 padding,视觉上"贴顶"
- `h-9` = 36px,跟 macOS titlebar 高度视觉对齐
- 拖动条下方才是真正的页面内容,内容 button / link 默认 no-drag

### Tauri 窗口配置

无需改 `tauri.conf.json`。`titleBarStyle: Overlay` + `hiddenTitle: true` 已经预留 native traffic light,跟 web 侧 drag region 不冲突。

## 风险 + 回滚

| 风险 | 回滚 |
|---|---|
| drag handle 视觉区太窄,用户拖不到 | 调 `h-9` → `h-10` / `h-11` |
| macOS native traffic light 视觉让位不够 | Sidebar `mt-10` → `mt-12` 临时调,后续 spec 化 |
| 回归:某路由页面的 page header 之前自己有 drag region 设计 | 后续路由检查;本 plan 只动 layout 层 |

## 不做

- **不**退回 V0.1.1 NSWindow 方案(违背 V0.1.4 retro 决策)
- **不**给所有交互子元素显式 `data-tauri-no-drag`(工作量大,本质仍是"整窗 drag"语义,本 plan 要放弃)
- **不**改 Sidebar mt-10 / Main mt-3 的让位间距(traffic light 让位是 V0.1.1 视觉决策,本 plan 不动)
- **不**改 `tauri.conf.json`(配置已就位)
