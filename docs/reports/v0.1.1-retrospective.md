# V0.1.1 Retrospective — 全局可拖的 L3 遗漏

> 创建: 2026-06-21
> 触发: V0.1.4 分析发现 setMovableByWindowBackground 副作用后回溯

## 1. Initial Decision(当初怎么决定的)

### 目标

V0.1.0 玻璃感被反馈"塑料感",V0.1.1 三件事并行:
- CSS 玻璃 token(`--glass-fill/border` 百分比)
- 透明窗口(`transparent: true` + `titleBarStyle: Overlay` + `hiddenTitle: true`)
- macOS native drag(`NSWindow.setMovableByWindowBackground: BOOL::YES`)

### 范围

- `src/index.css` token
- `src-tauri/tauri.conf.json` 窗口配置
- `src/routes/StyleGuideLayout.tsx` 玻璃容器布局
- `src-tauri/src/lib.rs` setup hook 调 objc2

### 验证手段

"能拖动 + 视觉对了 = smoke pass",用户口头确认。

## 2. Unidentified at Decision Time(当时没识别什么)

### L1 原始权威 — 读了,但没读完

读了 NSWindow 文档的 `setMovableByWindowBackground` 主条目,**没读 side effects 段**:
> "When set to YES, the window's background can be used to move the window. This includes text selection, click handling, and cursor changes that would normally occur in those areas."

这段明确警告 mousedown / cursor 会被拦截,但当时只确认了"能拖"就 ship。

### L2 统一设计 — 不适用

当时没有 drag 相关的 spec(spec 在 V0.1.2 才完整恢复)。

### L3 具体问题 — **严重欠枚举** ⭐ 漏 Layer 3

当时 L3 列表只有:
- ✅ "窗口能拖动"(L3 主路径)
- ✅ "按钮可点击"(WKWebView 拦截,当时验证过)
- ❌ **没列**:"文字能否被选中"
- ❌ **没列**:"窗口能否 resize"
- ❌ **没列**:"cursor 在 button 上是否变 pointer"

**根因**:smoke 模式是"打开 → 拖动 → 关闭",**没覆盖 user-facing 的副作用**。

## 3. Later Exposure(V0.1.4 才暴露的症状)

### 症状 1:文字无法选中

用户报告:在 CodeBlock 里想复制一段示例代码,**按鼠标拖动框选 → 窗口跟着飞,文字没被选中**。

### 症状 2:窗口右下角 resize 失败

`tauri.conf.json` 设了 `resizable: true`,但右下角 resize handle 区域也被 NSWindow 接管为 drag,**resize 行为分裂**。

### 症状 3:Copy button hover 无 cursor: pointer

用户精确 hover 在 CodeBlock 的 Copy 按钮上,**鼠标光标仍为箭头,不变手型**。button 元素虽然浏览器默认 cursor=pointer,但 NSWindow 抑制了 WKWebView 内的 cursor 更新。

### 真根因

**三个症状同一个根因**:`NSWindow.setMovableByWindowBackground: YES` 会拦截整个窗口背景的 mousedown / cursor 更新。修复 = 切换到 Tauri 2 官方 `data-tauri-drag-region`(CSS `webkit-app-region: drag`),只对显式标记的容器生效。

## 4. Generalized Lesson(可推广规则)

> **任何 macOS / 系统级 API 的 smoke,必须主动枚举 user-facing interaction layer 的副作用(text select / resize / cursor / hover / focus / keyboard),不能只验证 API 主行为。**

未来遇到同类场景(系统级 API、跨平台行为、WKWebView 与 NSWindow 交互),pre-commit checklist 必须包含:

- [ ] 文字是否可选中
- [ ] 窗口 resize 是否正常
- [ ] cursor 在交互元素上是否变化
- [ ] hover / focus 视觉反馈是否正常
- [ ] 键盘交互是否正常
