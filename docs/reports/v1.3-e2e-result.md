## 🏗️ 架构落地

### 后端（Rust）

- **4 表 schema**：task (12 字段 + 4 索引) / idea (3 字段) / check_in (4 字段) / record (5 字段 + UNIQUE 约束)
- **业务层**：4 个源表函数 + 1 个 record 视图模块（4 个查询）= 8 个业务函数
- **状态机**：4 状态 + 5 不变量（L1-L5 全部测试覆盖）
- **Tauri commands**：17 个（8 task + 1 idea + 1 check_in + 5 record + 3 floating）
- **多窗口**：main + floating (320×36 fold / 360×280 expand)
- **跨平台**：macOS NSPanel（不抢焦 + 全空间 + transient）+ 其他平台 no-op

### 前端（React 19）

- **FloatingWindow**（4 组件）：FocusBlock（折叠 3 态） + ExpandState（展开 360×280） + ControlRow + SwitchDropdown
- **Hook 体系**：useActiveTask / useTick / useWindowPosition / useRecords
- **主窗时间线**：RecordTimeline 4 tab（全部/任务/灵感/打卡） + 自动刷新（record-updated 事件）
- **Liquid Glass**：rgba 0.28-0.42 + blur 32px + saturate 200% + 1px 白色内描边
- **CSS 隔离**：每组件独立 CSS 文件，避免并行冲突

### 跨平台

- **macOS**：objc/cocoa 提升为 NSPanel，⌘+Shift+Space toggle
- **Windows / Linux**：Tauri 内置 alwaysOnTop + skipTaskbar，Ctrl+Shift+Space toggle
- **自启动**：tauri-plugin-autostart（macOS LaunchAgent / Windows registry）

## 🐛 已知问题 / 待优化

1. **tauri-bridge.ts 类型扩展**（Task 19 引入）：`Record` 类型扩展了 `duration_ms` + `focus_started_at` 字段。已在 Task 13 通过新增 `recordGetActiveTask` command 解决（前端用这个方法拿完整 Task）。

2. **`list_switchable_excludes_active_and_done` 测试微调**（Task 9 实施者报告）：spec 假设的 `t1=active + t3=done` 模式违反 L1 不变量（无法同时 start 两个 task）。实施者改为 `pause+start` 模式。测试通过，逻辑正确。

3. **Cargo.lock 未提交**：项目 root `.gitignore` 第 28 行忽略 `src-tauri/Cargo.lock`。这是项目约定（regenerable）。

4. **CSS 拆分**：原计划单一 `floating.css`，实施时拆分为 `glass.css` + `focus-block.css` + `switch-dropdown.css` + `expand-state.css`，避免并行冲突。
