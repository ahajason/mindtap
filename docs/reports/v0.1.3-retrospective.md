### 唯一未识别 — **没有给 button 加 `cursor-pointer` 显式声明**

V0.1.3 audit 给 input/textarea 加了 `cursor-text`,给 button 的 disabled 加了 `cursor-not-allowed`,但**没**给 button 主元素加 `cursor-pointer`(虽然浏览器默认就有)。

**当时 rationale**:"button 元素自带 cursor: pointer,不需要显式"。**但 V0.1.4 在 Tauri 全局可拖场景下发现:NSWindow 会抑制 WKWebView 内 cursor 更新,即使元素自带 pointer 也无效**。

**漏 L1**:没考虑到 NSWindow drag region 跟 cursor 抑制的交互。当时没引入 setMovableByWindowBackground,但 V0.1.1 已经引入了,V0.1.3 没回溯检查。

## 3. Later Exposure

### V0.1.4 才暴露

- **全局可拖的 3 症状同根因**(V0.1.1 retro 详述)
- **Copy button cursor 无效** = V0.1.3 没回溯 V0.1.1 的系统 API 决策

### V0.1.3 自己内部 — 没暴露新症状

所有 9 个 commit 的改动范围都限定在局部组件,**没影响其他引用**(V0.1.4 分析时已 grep 确认零回归)。

## 4. Generalized Lesson

> **修复时不仅要查 spec,还要回溯上一个版本是否引入了影响本次修复的系统级 API**。本次"cursor audit"漏了 V0.1.1 引入的 NSWindow 抑制。

具体预防措施:
- [ ] 任何"补全元素行为"类的修复前,grep `NSWindow` / 系统级 API 引用,看是否有跟本次修复交互的副作用
- [ ] cursor / hover / focus 这类 user-perception 修复,必须在**当前真实运行环境下** smoke,不能只看 spec
