## 1. 背景与目标

### 1.1 项目定位

轻念 · Mindtap — 极简记录桌面应用。技术栈:Tauri 2 (Rust) + React 19 + TypeScript + Vite 7。本地 SQLite,无云同步。

设计语言:浅色玻璃拟态(Light Glassmorphism),冷蓝调,3-tier 玻璃等级(L1/L2/L3)。

### 1.2 阶段状态

- V1.0 业务代码已于 2026-06-19 归档到 `.archive/`
- V0.1.0 起步:工程骨架就位(Tauri + React + Tailwind 4),业务代码待接入
- V0.1.0 = **通用组件样式演示**,**不**含任何业务组件

### 1.3 核心约束(不讨论)

1. 不引用 V1.0 业务代码 — `.archive/` 仅作历史教训参考
2. 不引用 V1.0 业务 spec — V0.1.0 的 spec 要新写
3. 起步阶段只写通用组件 + 设计规范,不引用 `.archive/` 业务细节

## 4. 设计文档三件套

### 4.1 文件清单与角色

```
    ├── README.md          ← reference only banner
    └── apple-hig/         ← cp 自 .archive/docs/material/apple/(按需,不批量)
```

### 4.2 spec 来源

字面 cp `.archive/docs/design/glassic-ui-spec.md`,**正文一字不改**。改动仅限:
- 顶部 frontmatter 元信息
- 关联文档区(参见 §3.2)

### 4.3 token 表(4 块内容)

1. **Tailwind 4 @theme block** — 把 spec 全部色值/模糊/圆角/字号落成 CSS 变量
2. **liquid-glass-react props 映射** — L1/L2/L3 对应 blur / tintOpacity / saturation / displacement
3. **状态色 → CVA 变体** — Button primary/secondary/ghost/icon 的 Tailwind class
4. **spec §八 强制项的代码侧检查** — grep / manual review 工具

### 4.4 component-format(11 组件)

跨组件共用规则(顶部章节):

| 规则 | 内容 |
|---|---|
| 命名 | 文件 `kebab-case.tsx`,导出 `PascalCase`;每个文件默认导出组件 |
| 类型 | Props 用 `interface`(不用 `type`),继承原生 HTML attrs |
| className | **总是**接受 `className` prop,通过 `cn()` 合并到末尾(允许外部覆盖) |
| ref 转发 | 所有 ui 组件用 `React.forwardRef`(Tabs / Tooltip / Dialog 由 Base UI 处理) |
| 空依赖 | 不依赖任何业务 store / context;纯展示组件 |
| glass 原则 | 组件本身不直接调 `backdrop-filter`,由父级 `<Glass>` 提供;实色组件(Button / Input)不需要 |
| i18n | V0.1.0 硬编码中文,V0.2+ 引入 i18n |
| **smoke 测试** | **每个组件配套 `*.test.tsx`**,验证: 挂载不报错 + 主要 props 默认值渲染 + 关键 variant 切换可见。 **不**做交互 / a11y / snapshot。 `npm test` 必须通过 |

## 6. 路由与导航

### 6.1 react-router-dom vs Tauri 多 Webview

| 形态 | 是什么 | V0.1.0 用? |
|---|---|---|
| **A. SPA + react-router-dom** | 一个 Webview 装一个 React 应用,URL 变化由 React Router 处理 | ✅ V0.1.0 主窗口 |
| **B. Tauri 多 Webview (`WebviewWindow`)** | 多个独立 OS 窗口,各自 Webview 进程 | ❌ V0.2+ 浮窗 / 设置窗口 |
| **C. MPA 多 HTML** | 多个独立 HTML,Webview 跳 URL | ❌ 极少用 |

**V0.1.0**: 装 `react-router-dom@^7`,主窗口单 SPA。  
**V0.2+**: 浮窗走 `WebviewWindowBuilder`,窗口间通信用 Tauri `emit`/`listen` + `invoke`。

### 6.2 唯一新依赖

| 包 | 动作 | 理由 |
|---|---|---|
| `react-router-dom` | **新装** ^7 | style guide 侧栏路由必需,React 19 兼容版 |
| 其他 | 不动 | liquid-glass-react / @base-ui / shadcn / sonner / cva 全部复用 |

### 8.2 详写 — Button

**Props API**
```ts
type ButtonVariant = 'primary' | 'secondary' | 'ghost' | 'icon';
type ButtonSize = 'sm' | 'md';
interface ButtonProps
  extends Omit<React.ButtonHTMLAttributes<HTMLButtonElement>, 'size'> {
  variant?: ButtonVariant;   // default: 'primary'
  size?: ButtonSize;         // default: 'md'
}
```

**Variants (cva)**
| variant | size | 视觉(spec §五.1) | 用途 |
|---|---|---|---|
| primary | md | 36-44px, 主色渐变 + 软阴影, 白字 | 主操作(每页最多 1) |
| primary | sm | 32px, 同上缩小 | 紧凑主操作 |
| secondary | md | 40px, L1 玻璃, `#475467` 字 | 次操作 |
| secondary | sm | 32px, 同上 | 紧凑次操作 |
| ghost | md | 40px, 透明背景, hover L1 | 第三级 / 工具 |
| icon | md | 36×36 圆形, L1 玻璃 | 工具栏 |

**Do / Don't**
- ✅ 一个页面最多 1 个 primary
- ❌ 不要 ghost 装成次按钮
- ❌ 不要 icon variant 装文字

**依赖** — `cn`;`lucide-react`(icon 槽);❌ 不依赖 `liquid-glass-react`(Button 本身实色)。

### 8.3 详写 — Card

**Props API**
```ts
type CardTier = 'l1' | 'l2' | 'l3';
interface CardProps extends React.HTMLAttributes<HTMLDivElement> {
  tier?: CardTier;     // default: 'l1'
}
```

**Variants**
| tier | 模糊 | 填充 | 描边 | 阴影 | 圆角 | 用途 |
|---|---|---|---|---|---|---|
| l1 | 20px | 0.35 | 0.60 | 0.08 | 20px | 普通卡片 / 标签 |
| l2 | 24px | 0.42 | 0.70 | 0.10 | 24px | 输入框 / 次按钮容器 |
| l3 | 28px | 0.50 | 0.80 | 0.12 | 28px | 主按钮 / 悬浮弹窗 |

**Do / Don't**
- ✅ 用 tier 表达 Z 轴层级,不要写自定义 boxShadow
- ❌ 不要在 Card 内再放 glass 子组件(避免叠层模糊性能问题)
- ❌ 不要 l3 用作"普通背景"

**依赖** — `cn`;`liquid-glass-react`(`<Glass />` 包 Card 外层)。

### 8.4 详写 — Tabs

**Props API**
```ts
import { Tabs } from '@base-ui/react/tabs';
// 直接用 @base-ui/react 的 Tabs.Root / Tabs.List / Tabs.Trigger / Tabs.Panel
```

**结构**
```tsx
<Tabs.Root defaultValue="preview">
  <Tabs.List>
    <Tabs.Trigger value="preview">预览</Tabs.Trigger>
    <Tabs.Trigger value="code">代码</Tabs.Trigger>
    <Tabs.Trigger value="props">属性</Tabs.Trigger>
  </Tabs.List>
  <Tabs.Panel value="preview">...</Tabs.Panel>
  <Tabs.Panel value="code">...</Tabs.Panel>
  <Tabs.Panel value="props">...</Tabs.Panel>
</Tabs.Root>
```

**Do / Don't** — 演示页统一用 Tabs 切三段;❌ 业务页不要硬塞 "预览/代码/属性" 结构。

**依赖** — `@base-ui/react/tabs`;`lucide-react`(激活态小图标)。

### 8.5 8 个紧凑组件(速查表)

| 组件 | Props | Variants | 依赖 |
|---|---|---|---|
| **Input** | `{ size?: 'sm'\|'md'; error?: boolean; ...InputHTMLAttributes }` | sm(36px) / md(40px);error 描边切红色 | cn, lucide-react |
| **Textarea** | `{ size?: 'sm'\|'md'; error?: boolean; ...TextareaHTMLAttributes }` | sm(2 行) / md(4 行);同 Input | cn |
| **Label** | `{ required?: boolean; ...LabelHTMLAttributes }` | 单一;required 显示 `*` | cn |
| **Badge** | `{ variant?: 'default'\|'success'\|'inactive'\|'warning' }` | 4 状态色,玻璃 L1 背景 | cn, lucide-react |
| **Separator** | `{ orientation?: 'horizontal'\|'vertical' }` | 1px 玻璃高光描边 | cn |
| **Tooltip** | `@base-ui/react` Tooltip,包一层统一 side prop | top / right / bottom / left | @base-ui/react/tooltip |
| **Dialog** | `@base-ui/react` Dialog | (Base UI 已提供) | @base-ui/react/dialog |
| **Sonner** | re-export `<Toaster />`,全局 theme='light',richColors | (V0.1.0 单主题) | sonner |
