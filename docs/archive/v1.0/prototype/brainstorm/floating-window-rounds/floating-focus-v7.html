浮动窗口形态 v7 — "进行中"的视觉探索

核心洞察：进行中 ≠ TODO。给"当前专注"换一种视觉语言 —— 标签名 / 形态 / 状态指示都要变

🎯 核心洞察

TODO = "待办" = 静态清单项
进行中 = "Focus / Now / 当前专注" = 动态活动，有时间在流动
→ 视觉上要 摆脱 TODO 的"清单感"，给"进行中"独立的视觉语言：
  · 状态标签改名（Focus / Now / 专注 / 进行中）
  · 视觉指示强化（呼吸灯 / 进度环 / 状态色）
  · 形态差异化（圆 / 环 / 流动感）

1

Focus 标签 + 呼吸灯

320×36px。绿色"Focus"标签 + 任务 + 计时 + ⏸。呼吸灯动画表示"正在活跃"。极简但状态感强。

① 折叠 · 进行中（Focus）36px
FOCUS 写 Q3 计划 00:12:34 ⏸
① 对照 · 折叠 · 未开始（普通 TODO）
TODO 3 个未开始 点击新建 ›

Pros

  • 呼吸灯动画表示"活跃中"，状态感最强
  • "Focus"标签替代 TODO，语义清晰
  • 未开始时切换为静态 TODO 样式，区分明显

Cons

  • 动画可能耗电（但 1.6s 周期很轻）
  • 绿色 + 蓝色双色搭配，需确认不冲突

⏸
✓ 完成 ↻ 切换
✓ 待办 ✎ 灵感 ⚡ 打卡
⏎ 保存并开始 ⏵

Pros

  • 进度环视觉化"截止时间"概念，时间感强
  • 绿色 FOCUS 标签 + 绿环，状态色统一
  • 环的"完成度"隐喻进行中，进度可见

Cons

  • 需要"截止时间"概念，简化版可不做进度
  • 环占空间（32px），任务名要短
  • 实现复杂度高（SVG 进度动画）

  • NOW 偏英文，中文环境可能不习惯

v7 状态机说明

未开始 (TODO)：灰色静态 pill + 普通任务列表样式
进行中 (Focus / Now / 进行中)：绿色 active 标签 + 状态动画（脉冲/进度/流动）+ 计时醒目
完成：归档到主应用，浮动窗不再显示
核心：进行中 ≠ TODO，必须有视觉差异化（否则用户分不清"想做的事"和"正在做的事"）

4 个"进行中"视觉方案 → 选 1 / 2 / 3 / 4（或多选"X 为主，Y 元素借鉴"）。
