# 浅色玻璃拟态（Light Glassmorphism）— 开发落地实现规范

> **定位**：本项目视觉设计语言的**强制实现规范**。所有前端组件的样式、UI 评审、PR Review 必须对照本规范。Apple HIG 参考资料在 [`docs/material/apple/`](../../material/apple/)；本文件是落到本项目 CSS 层的具体参数与色值。

---

## 一、整体视觉体系

核心为 **浅色玻璃拟态（Light Glassmorphism）**，对齐 macOS 原生磨砂玻璃设计语言，整体呈冷调浅蓝清透质感。通过「**背景模糊 + 半透填充 + 高光描边 + 软阴影**」四层材质构建 Z 轴悬浮分层效果，无多余装饰元素，视觉轻盈、克制，符合生产力工具的高效属性。

---

## 二、核心玻璃材质实现标准（必选四要素）

> 所有卡片、输入框、次级按钮等容器组件，**必须同时满足**以下 4 个属性，缺一不可，否则会丢失真实玻璃质感。

| 属性 | 参数规范 | 作用说明 |
| :--- | :--- | :--- |
| 背景模糊 | `backdrop-filter: blur(20px) saturate(120%)`<br>需补充 `-webkit-backdrop-filter` 兼容 Safari | 质感核心：对元素后方内容做磨砂模糊；`saturate` 轻微提升饱和度，避免效果发灰发闷 |
| 半透填充 | `background: rgba(255, 255, 255, 0.35 ~ 0.5)`<br>组件层级越高，填充不透明度越高（差值 ≈ 10%） | 模拟玻璃基材的白色雾感，平衡透光性与内容可读性 |
| 边缘高光描边 | `border: 1px solid rgba(255, 255, 255, 0.6 ~ 0.8)` | 模拟玻璃边缘的光线折射，明确组件边界，是区分「塑料感」和「玻璃感」的关键细节 |
| 软质外阴影 | `box-shadow: 0 8px 32px rgba(0, 30, 80, 0.08 ~ 0.12)` | 营造悬浮立体感；阴影必须低不透明度、大模糊半径，禁止硬边阴影 |

**分层逻辑**：顶层组件（主按钮、悬浮弹窗）→ 模糊半径更大、填充不透明度更高、阴影更深；中层次级卡片参数依次降低，构建清晰的空间纵深。

### 层级参数对照表

| 层级 | 模糊半径 | 半透填充 | 高光描边 | 软阴影 |
| :--- | :--- | :--- | :--- | :--- |
| **L1 基础**（普通卡片 / 标签） | `blur(20px)` | `rgba(255, 255, 255, 0.35)` | `rgba(255, 255, 255, 0.60)` | `0 8px 32px rgba(0, 30, 80, 0.08)` |
| **L2 中等**（输入框 / 次按钮） | `blur(24px)` | `rgba(255, 255, 255, 0.42)` | `rgba(255, 255, 255, 0.70)` | `0 8px 32px rgba(0, 30, 80, 0.10)` |
| **L3 顶层**（主按钮 / 悬浮弹窗） | `blur(28px)` | `rgba(255, 255, 255, 0.50)` | `rgba(255, 255, 255, 0.80)` | `0 8px 32px rgba(0, 30, 80, 0.12)` |

> 三层之间：填充差值 ≈ `10%`、描边差值 ≈ `10%`、阴影差值 ≈ `0.02`，符合分层逻辑。

### 参考 SCSS 片段

```scss
@mixin glass-surface($level: 1) {
  $blur:   if($level >= 3, 28px, if($level >= 2, 24px, 20px));
  $fill:   if($level >= 3, 0.50, if($level >= 2, 0.42, 0.35));
  $border: if($level >= 3, 0.80, if($level >= 2, 0.70, 0.60));
  $shadow: if($level >= 3, 0.12, if($level >= 2, 0.10, 0.08));

  backdrop-filter: blur($blur) saturate(120%);
  -webkit-backdrop-filter: blur($blur) saturate(120%);
  background: rgba(255, 255, 255, $fill);
  border: 1px solid rgba(255, 255, 255, $border);
  box-shadow: 0 8px 32px rgba(0, 30, 80, $shadow);
}
```

## 三、色彩系统（可落地色值区间）

整体为低饱和冷蓝灰调色，无强对比杂色，色值可直接用于开发。

### 1. 背景层

- **基底渐变**：`#F5F9FF` → `#E8F1FF` 柔和径向 / 线性渐变
- **细节纹理**：叠加柔化流体光带，局部点缀极淡虹彩色散（RGB 通道微偏移，不透明度 ≤ 10%），仅作质感点缀，不可突出

### 2. 主强调色

- 色值参考 `#165DFF`（宝蓝色域）
- **仅用于**：主按钮、核心数值、激活状态

| 状态 | 色值 / 处理 | 用途 |
| :--- | :--- | :--- |
| **default** | `#165DFF` | 默认 |
| **hover** | `#0E4AD9`（或 `#165DFF` × 92% alpha） | 鼠标悬停 |
| **active / pressed** | `#0A3DBC`（或 `#165DFF` × 85% alpha） | 按下瞬间 |
| **disabled** | `#165DFF` × 40% alpha | 禁用 |
| **明度渐变** | `linear-gradient(180deg, #165DFF 0%, #0E4AD9 100%)` | 主按钮背景 |
| **发光阴影** | `0 4px 12px rgba(22, 93, 255, 0.25)` | 主按钮默认阴影 |

3.  **文本色阶（满足 WCAG AA 对比度）**
    - 一级标题：`#1D2939`（深灰蓝）
    - 二级说明：`#475467`（中灰蓝）
    - 三级提示：`#98A2B3`（浅灰蓝）
4.  **功能状态色**
    - 完成 / 成功：低饱和薄荷绿；未激活：浅灰；均为柔和平涂，无高饱和刺眼色

| 状态 | 色值 | 备注 |
| :--- | :--- | :--- |
| **成功 / 完成** | `#5BCBA0`（低饱和薄荷绿） | 平涂，无描边 |
| **未激活** | `#DDE3EE`（冷调浅灰） | 仅文字 / 占位 |
| **警告**（如需） | `#F5A623` × 80% alpha（低饱和橙） | 慎用 |
| **错误**（如需） | `#E5484D` × 85% alpha（低饱和红） | 慎用 |

## 四、形态与圆角规范

全界面统一大圆角体系，无尖锐直角，圆角半径随容器尺寸等比匹配：

- 大型功能卡片：圆角半径 `20px ~ 28px`
- 中型输入框、次级按钮：圆角半径 `12px ~ 16px`
- 小型控件（图标容器、状态点）：全圆角（圆形）
- 所有图标线条、端点、转角均为圆角处理，线条粗细均匀（约 2px）

| Token | 值 | 用途 |
| :--- | :--- | :--- |
| `space-1` | `4px` | 极小间距（图标与文字） |
| `space-2` | `8px` | 紧凑间距（控件内部） |

## 五、组件通用规则

### 1. 按钮体系

- **主按钮**：实色填充（主强调色），叠加自上而下的轻微明度渐变，白色图标 / 文字，搭配软阴影；点击态做 `0.96 ~ 0.98` 微缩放
- **次按钮 / 标签按钮**：玻璃拟态材质（同卡片规范 §二），深灰图标 / 文字；激活态提升填充不透明度，文字切换为主强调色

## 六、排版与层级

- 字体：采用无衬线体系（类 SF Pro / Inter），字重层级清晰
- 字号梯度：
  - 核心数值（如计时数字）：**超大字号 + 粗字重 + 主强调色**，是单模块视觉焦点
  - 标题、说明、辅助文字：字号 / 字重依次递减
- 全局留白充足，对齐逻辑严谨，无拥挤堆砌

## 七、交互与动效

- 缓动曲线：`ease-out` 柔和过渡，动效时长 `200ms ~ 300ms`，无生硬弹跳
- 状态切换：以「透明度变化 + 微缩放 + 轻微位移」为主，过渡顺滑
- 悬停反馈：玻璃组件悬停时，填充不透明度提升 `5% ~ 10%`，阴影模糊半径轻微增大，反馈克制

### 缓动曲线 token

| Token | 值 | 用途 |
| :--- | :--- | :--- |
| `ease-out` | `cubic-bezier(0.2, 0.8, 0.2, 1)` | 标准缓出（默认） |
| `ease-in-out` | `cubic-bezier(0.4, 0, 0.2, 1)` | 进出场（弹窗） |

### 动效时长 token

| Token | 值 | 用途 |
| :--- | :--- | :--- |
| `duration-fast` | `150ms` | 状态微调（hover 颜色） |
| `duration-base` | `240ms` | 默认（按钮按下、状态切换） |
| `duration-slow` | `320ms` | 弹窗进出 / 抽屉 |

### 状态切换位移 / 缩放参数

| 动作 | 值 |
| :--- | :--- |
| 微缩放 | `scale(0.96 ~ 0.98)`（按下态） |
| 轻微位移 | `translateY(-1px ~ -2px)`（hover 上抬） |
| 透明度变化 | `opacity 0 → 1`（进出） |
| 阴影模糊半径增大 | `32px → 40px`（hover 强化） |
| 填充不透明度提升 | `+5% ~ +10%`（hover 反馈） |

## 八、高频实现偏差避坑（强制执行）

1.  **禁止用"半透明白色"替代"背景模糊"**：玻璃感的核心是后方内容的模糊透光，仅做纯色半透会呈现塑料 / 雾面感，必须启用 `backdrop-filter`
2.  **禁止高不透明度硬阴影**：阴影必须是大半径、低不透明度的弥散效果，否则会显脏、显厚重，破坏轻盈感
3.  **色散效果必须极淡**：虹光仅作局部细节增强，饱和度和不透明度严格压低，避免廉价感
4.  **保障文本对比度**：玻璃背景上的正文必须满足 WCAG AA 4.5:1 对比度，不可为了质感牺牲可读性
5.  **背景必须有层次**：纯色纯白背景无法体现玻璃拟态效果，必须保留渐变 + 光影纹理的底层内容

> 上述 5 条为**强制项**，PR Review 时任一违反需打回修改。
