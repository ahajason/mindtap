## 7. 经验沉淀（v1.5 借鉴）

- **subagent-driven-development skill 对 spec-driven 任务极有效**：每 PR brief 给完整代码 + base SHA → subagent transcription + test running ≈ 100% 命中。PR10 / PR11 各发现 1 个 reviewer-only 发现（onCollapse 顺序 / floating_toggle 副作用），证明 review loop 不是仪式。
- **inline fix > dispatch subagent** for trivial 1-3 行修改（PR8 fix / PR10 fix / PR11 fix 都是 inline）：节省 ~30k tokens/次 dispatch。
- **保留 SPEC 决策的 "v1 不做" vs "dead code" 区分**：RecordTimeline 0 import 但 spec 决策 21 保留——别因为 grep 不到 import 就删，未来 sprint 可能复用。
