# P3.5 脚手架卫生 + 工具链配置

> **背景**：V1.0 P3 脚手架（commit `64ed8e3`）已通过最小跑通验收（WSL dev + Windows build .exe + 1 Surface demo）。
> 脚手架本身**结构符合 Tauri 2 主流惯例**，但身份字段（productName / package name / Cargo name）全是 Tauri 默认值「tauri-app」，
> 且缺少开发者侧的工程规范（路径别名、Node 版本锁、editorconfig、rustfmt）。本 spec 收尾这两类问题。

---

## 1. 目标

在**不动业务代码、不动目录结构、不加新依赖**的前提下，把脚手架从「能跑」推到「可协作 + 可发布」状态。

## 2. 非目标（明确边界）

- 不动 `App.tsx` / `Surface.tsx` / `glass.css`（业务资产）
- 不动 `src/` + `src-tauri/` 二级结构（Tauri 官方默认）
- 不加 react-router / zustand / tauri-plugin-store（属 P4 业务）
- 不动 vite.config.ts 现有 `@ts-expect-error` 注释（跨平台 dev 需要）
- 不动 `tsconfig.json` 严格度（已 strict + noUnused）
- 不动 `CSP: null`（dev 默认即可）
- 不动 prototype/ 内现有原型（保留作 V1 视觉决策资产）

## 3. 范围

11 个模块分 2 批 commit：

| 批次 | 主题 | 模块 | 文件数 | 估时 |
|---|---|---|---|---|
| **A** | identity-first | 1-7 | 8 | ~30 min |
| **B** | tooling | 8-11 | 5 | ~25 min |

> 理由：npm registry 标识、`tauri.conf.json` 引用名、GitHub 显示来源。

#### 模块 2: `src-tauri/Cargo.toml` — crate 元数据

- `name`: `"tauri-app"` → `"mindtap"`
- `authors`: `["you"]` → `["AhaJason <Jason-c1@outlook.com>"]`
- `description`: `"A Tauri App"` → `"极简记录 APP · 跨平台桌面"`

#### 模块 3: `src-tauri/tauri.conf.json` — productName / window / bundle

- `productName`: `"tauri-app"` → `"轻念 · Mindtap"`

- `app.windows[0].label`: 加 `"label": "main"`（Tauri 2 多窗口/菜单场景必须显式）

> 理由：productName 影响安装包名 + 窗口标题 + dock tooltip；`bundle.targets: "all"` 会触发 Tauri 在 build 时拉取 NSIS/WiX 工具链检测（V1.0 不打安装包，应显式空数组关闭）；窗口 label 在 Tauri 2 是必填默认字段，主流做法显式声明。

- `<html lang="en">` → `<html lang="zh-CN">`
- `<title>Tauri + React + Typescript</title>` → `<title>轻念 · Mindtap</title>`
- 其它字段不动

> 理由：`lang` 影响浏览器拼写检查、屏幕阅读器发音、字体 fallback。`<title>` 是 OS 任务栏/标签页显示文本。

### Batch B — tooling（commit 2）

#### 模块 8: tsconfig + Vite 路径别名 `@/*`

`tsconfig.json` 加 `paths`：

- `compilerOptions.baseUrl`: `"."`
- `compilerOptions.paths`: `{ "@/*": ["src/*"] }`

`vite.config.ts` 加 `resolve.alias`：

- `import path from "node:path"`
- `import { fileURLToPath } from "node:url"`
- `const __dirname = fileURLToPath(new URL(".", import.meta.url))`
- `resolve: { alias: { "@": path.resolve(__dirname, "src") } }`
- 去掉 `// @ts-expect-error process is a nodejs global` 注释（vite.config 不再用 `process.env`）

试用：把 `src/App.tsx` 的 `import './styles/glass.css'` 改为 `import '@/styles/glass.css'`，
       `src/App.tsx` 里如果引用 `./components/Surface` 也改为 `@/components/Surface`。

> 理由：tsconfig `paths` 给 TS 编译器看（编译时），Vite `resolve.alias` 给打包器看（运行时），两者必须同步。
> 主流做法：`@/*` 映射到 `src/*` 是 Vite/Next/Remix 社区事实标准。关键陷阱：Vite ESM 模式下无 `__dirname`，要用 `fileURLToPath` 转换。
> **否则配了不用就是死配置** — 至少改 1 处 import 验证。

#### 模块 9: Node 版本锁

- 新增 `.nvmrc`：内容为 `24`（不带 `v` 前缀，不带换行外内容）
- `package.json` `engines.node`：`">=24 <25"`（同时给出 min/max）

> 理由：项目当前 Node 24（WSL + Windows 一致），但未固化。`.nvmrc` 跨平台事实标准；`engines` 给 npm 装包时校验。
> 关键：`engines` 写宽 `>=24 <25` 而不是 `24.x` 是为 patch 版本自动跟随。

#### 模块 10: `.editorconfig`

新增根目录 `.editorconfig`：

- `root = true`
- `[*]` 通配规则：`charset = utf-8`、`end_of_line = lf`、`indent_style = space`、`indent_size = 2`、`trim_trailing_whitespace = true`、`insert_final_newline = true`
- `[*.md]` 例外：`trim_trailing_whitespace = false`（行尾空格在 Markdown 里有意义）

> 理由：跨编辑器（VS Code / Zed / Vim / Cursor）保持一致格式。
> 关键：`end_of_line = lf` — WSL/Linux 默认；Windows 工具链 CRLF 写出会污染 git diff。

#### 模块 11: `src-tauri/rustfmt.toml`

新增 `src-tauri/rustfmt.toml`：

- `edition = "2021"`
- `max_width = 100`
- `tab_spaces = 2`

> 理由：Rust 风格与 JS 不同（max_width 100 vs Prettier 80 是社区共识），rust-analyzer 自动读取。
> 关键：放在 `src-tauri/` 内（不影响其它项目；cargo 自动发现）。

## 5. Commit 策略

| # | 消息 | 文件 |
|---|---|---|
| 1 | `chore(p3.5): 重命名项目身份 + 同步元数据 + 清理空目录` | 8 文件（M/D + .gitignore 改 1 行） |
| 2 | `chore(p3.5): 工具链配置 — 路径别名 / Node 锁 / editorconfig / rustfmt` | 6 文件（3 新增 + 3 改：tsconfig / vite.config / App.tsx 改；.nvmrc / .editorconfig / rustfmt.toml 新增） |

**分两个 commit 的理由**：
- Commit 1 是**产品身份**（用户/打包机/任务栏可见）
- Commit 2 是**开发者规范**（编辑器/TS/Rust 格式化）
- 未来回滚可独立回退一个维度

## 7. 回滚

| 场景 | 动作 |
|---|---|
| 单文件出错 | `git checkout HEAD~1 -- <file>` |
| 整个 commit 错 | `git revert HEAD`（保留历史）或 `git reset --hard HEAD~1`（未 push 时） |
| 工具链配置误伤 IDE | 删 `.editorconfig` / `src-tauri/rustfmt.toml` 即可关闭 |
| 路径别名循环依赖 | 删 `tsconfig.json` `paths` + `vite.config.ts` `resolve.alias` 即可恢复 |
