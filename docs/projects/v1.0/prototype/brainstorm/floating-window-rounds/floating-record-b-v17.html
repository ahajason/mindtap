D4-B 业务层双写 · 事务包裹

核心原则

  • 原子性：源表 INSERT + record INSERT 必须同事务，全部成功或全部回滚
  • source_id 来源：源表 INSERT 后取 last_insert_rowid()，再写 record
  • UPDATE 同步：task content/status 变更时，业务层同步 UPDATE record
  • DELETE 级联：业务层先删 record，再删源表（或反过来包事务）

Rust 业务层示例（事务双写）

// src-tauri/src/db.rs

use rusqlite::{Connection, Transaction, params};
use serde::{Serialize, Deserialize};

#[derive(Debug, Serialize, Deserialize)]
pub struct Task {
    pub id: i64,
    pub content: String,
    pub status: String,
    pub duration_ms: i64,
    pub first_focused_at: Option,
    pub focus_started_at: Option,
    pub paused_at: Option,
    pub focused_count: i64,
    pub due_at: Option,
    pub created_at: i64,
    pub updated_at: i64,
    pub archived_at: Option,
}

/// 业务层：创建 task（双写 record）
pub fn create_task(
    conn: &mut Connection,
    content: &str,
    due_at: Option,
) -> rusqlite::Result {
    let now = now_ms();

    // 1. 开事务
    let tx = conn.transaction()?;

    // 2. INSERT 源表
    tx.execute(
        "INSERT INTO task (content, status, created_at, updated_at, due_at)
         VALUES (?1, 'pending', ?2, ?2, ?3)",
        params![content, now, due_at],
    )?;
    let task_id = tx.last_insert_rowid();

    // 3. INSERT 通用 record（双写）
    tx.execute(
        "INSERT INTO record (kind, source_id, content, status, created_at)
         VALUES ('task', ?1, ?2, 'pending', ?3)",
        params![task_id, content, now],
    )?;

    // 4. 提交
    tx.commit()?;

    // 5. 返回
    Ok(Task { id: task_id, content: content.to_string(), status: "pending".to_string(),
              duration_ms: 0, first_focused_at: None, focus_started_at: None,
              paused_at: None, focused_count: 0, due_at,
              created_at: now, updated_at: now, archived_at: None })
}

/// 业务层：更新 task status（双写 record.status）
pub fn update_task_status(
    conn: &mut Connection,
    task_id: i64,
    new_status: &str,
) -> rusqlite::Result<()> {
    let now = now_ms();
    let tx = conn.transaction()?;

    // 1. UPDATE 源表
    tx.execute(
        "UPDATE task SET status = ?1, updated_at = ?2 WHERE id = ?3",
        params![new_status, now, task_id],
    )?;

    // 2. UPDATE 通用 record
    tx.execute(
        "UPDATE record SET status = ?1 WHERE kind = 'task' AND source_id = ?2",
        params![new_status, task_id],
    )?;

    tx.commit()?;
    Ok(())
}

/// 业务层：删除 task（双写 record 级联）
pub fn delete_task(conn: &mut Connection, task_id: i64) -> rusqlite::Result<()> {
    let tx = conn.transaction()?;
    tx.execute("DELETE FROM record WHERE kind = 'task' AND source_id = ?1", params![task_id])?;
    tx.execute("DELETE FROM task WHERE id = ?1", params![task_id])?;
    tx.commit()?;
    Ok(())
}

/// 业务层：创建 idea
pub fn create_idea(conn: &mut Connection, content: &str) -> rusqlite::Result {
    let now = now_ms();
    let tx = conn.transaction()?;
    tx.execute("INSERT INTO idea (content, created_at) VALUES (?1, ?2)",
               params![content, now])?;
    let id = tx.last_insert_rowid();
    tx.execute("INSERT INTO record (kind, source_id, content, status, created_at)
                VALUES ('idea', ?1, ?2, NULL, ?3)", params![id, content, now])?;
    tx.commit()?;
    Ok(id)
}

/// 业务层：创建 check_in
pub fn create_check_in(
    conn: &mut Connection,
    habit: &str,
    note: Option<&str>,
) -> rusqlite::Result {
    let now = now_ms();
    let tx = conn.transaction()?;
    tx.execute("INSERT INTO check_in (habit, note, checked_at) VALUES (?1, ?2, ?3)",
               params![habit, note, now])?;
    let id = tx.last_insert_rowid();
    let content = match note {
        Some(n) => format!("{} · {}", habit, n),
        None => habit.to_string(),
    };
    tx.execute("INSERT INTO record (kind, source_id, content, status, created_at)
                VALUES ('check_in', ?1, ?2, NULL, ?3)", params![id, content, now])?;
    tx.commit()?;
    Ok(id)
}

B 方案工程要点

要点做法
事务强制所有双写函数必须用 conn.transaction()，禁止裸 execute
source_id 来源tx.last_insert_rowid() 紧跟 INSERT
UPDATE 同步每次改 task.content/status 都同步 record
代码层抽象提供 create_* / update_* / delete_* 业务函数，业务层不直接 SQL
测试覆盖单元测试：每个 create_*/update_*/delete_* 都要验证 record 同步
lint 规则禁止业务层直接 INSERT INTO task（必须走业务函数）

3 3 关注点 · 推荐答案

① V1.2 → V1.3 数据迁移

背景：V1.2 用单表 record(id, type, content, payload, created_at, updated_at)，type ∈ {todo, idea, check_in}

目标：拆 3 源表 + 新通用 record 表，旧数据无损迁移

推荐迁移步骤

  1. 检测：V1.3 启动时检测是否存在 record 旧表（schema_version < 1.3）
  2. 建新表：执行 1.3 完整 schema（task/idea/check_in/record 新表）
  3. 迁移数据：
    • 旧 type='todo' → 拆到 task 表（status 默认 'pending'，duration_ms=0）→ 自动触发 record 写
    • 旧 type='idea' → 拆到 idea 表 → 自动触发
    • 旧 type='check_in' → 拆到 check_in 表（habit 字段从 content 提取）
  4. 回填 record：迁移期间业务层双写暂停，先 INSERT 源表，再统一回填 record
  5. 校验：迁移后 count 校验（旧 record 数 = 新 3 表数 = 新 record 数）
  6. 备份：迁移前 record_v12 备份表，30 天后自动 DROP

迁移 SQL 片段

-- 检测旧表
SELECT name FROM sqlite_master WHERE type='table' AND name='record_v12';

-- 迁移 todo
INSERT INTO task (id, content, status, created_at, updated_at)
SELECT id, content,
       CASE WHEN payload LIKE '%done%' THEN 'done' ELSE 'pending' END,
       created_at, updated_at
FROM record_v12 WHERE type = 'todo';

-- 迁移 idea
INSERT INTO idea (id, content, created_at)
SELECT id, content, created_at FROM record_v12 WHERE type = 'idea';

-- 迁移 check_in（habit 字段从 content 提取为默认 '未命名习惯'）
INSERT INTO check_in (id, habit, note, checked_at)
SELECT id, content, NULL, created_at FROM record_v12 WHERE type = 'check_in';

-- 触发器自动写 record，但迁移时业务层双写关闭 → 手动回填
INSERT INTO record (kind, source_id, content, status, created_at)
SELECT 'task', id, content, status, created_at FROM task
WHERE id NOT IN (SELECT source_id FROM record WHERE kind = 'task');
-- 同理 idea / check_in

③ 归档 task 是否从 record 隐藏

推荐 A · 隐藏归档

  • task.archived_at ≠ null → record 中隐藏
  • 主窗时间线默认不显示归档
  • 主窗"归档"独立 tab 可见（统计用）
  • 隐藏不是删除（可恢复）

备选 B · 保留显示

  • 归档也显示，标灰 + "已归档"标签
  • 列表更长，更完整
  • 不符合"重记录轻查看"原则

推荐 A：符合产品哲学（重记录轻查看，避免列表冗长），归档独立 tab 仍可访问。

归档 SQL

-- 主窗时间线（隐藏归档）
SELECT * FROM record
WHERE kind != 'task' OR source_id NOT IN (SELECT id FROM task WHERE archived_at IS NOT NULL)
ORDER BY created_at DESC;

-- 主窗"归档" tab
SELECT r.* FROM record r
JOIN task t ON r.kind = 'task' AND r.source_id = t.id
WHERE t.archived_at IS NOT NULL
ORDER BY t.archived_at DESC;
