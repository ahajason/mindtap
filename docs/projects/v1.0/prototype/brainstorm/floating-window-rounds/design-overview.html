痛点与设计铁律

🎯 目标用户

ENFP 创意型 — 点子爆发但极易遗忘、难以长期坚持记录

  • 同时处理 3-5 个并行任务，频繁切换
  • 灵感瞬间即逝，必须 3 秒内记录
  • 需要"轻推"（视觉提示）想起当前在做什么

📐 唯一设计准则

3 秒记录 / 1 秒查看 / 0 思考成本

  • 3 秒：快捷键 ≤ 1 击 + 输入 + 保存 1 键
  • 1 秒：折叠态 ≤ 36px 常驻任意位置
  • 0 思考：无 Tab 切换 / 无强制类型 / 智能推断

❌ V1.2 痛点

  • 像"应用程序后台"而非轻量 app — 打开主窗才看记录
  • 多维度记录操作繁琐（至少 5 步）— 灵感瞬间流失
  • TODO 状态机缺失 — 只有"完成/未开始"二态，无法表达"正在做"

数据模型 · 3 源表 + 1 通用 record 表

核心决策：拆分 3 源表（task/idea/check_in）+ 1 通用 record 表（业务层事务双写），主窗统一列表展示。

4.1 task 表（4 状态机）

CREATE TABLE task (
  id                  INTEGER PRIMARY KEY AUTOINCREMENT,
  content             TEXT    NOT NULL,
  status              TEXT    NOT NULL DEFAULT 'pending',
  duration_ms         INTEGER NOT NULL DEFAULT 0,
  first_focused_at    INTEGER,
  focus_started_at    INTEGER,
  paused_at           INTEGER,
  focused_count       INTEGER NOT NULL DEFAULT 0,
  due_at              INTEGER,
  created_at          INTEGER NOT NULL,
  updated_at          INTEGER NOT NULL,
  archived_at         INTEGER
);

4.2 idea / check_in / record 表

CREATE TABLE idea (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  content     TEXT    NOT NULL,
  created_at  INTEGER NOT NULL
);

CREATE TABLE check_in (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  habit       TEXT    NOT NULL,
  note        TEXT,
  checked_at  INTEGER NOT NULL
);

CREATE TABLE record (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  kind        TEXT    NOT NULL,        -- 'task' | 'idea' | 'check_in'
  source_id   INTEGER NOT NULL,
  content     TEXT    NOT NULL,
  status      TEXT,
  created_at  INTEGER NOT NULL
);

4.3 业务层事务双写（D4-B 锁定）

源表 INSERT + record INSERT 必须同事务，全部成功或全部回滚：

pub fn create_task(conn: &mut Connection, content: &str,
                   due_at: Option<i64>) -> rusqlite::Result<Task> {
    let now = now_ms();
    let tx = conn.transaction()?;                    // ← 开事务
    tx.execute("INSERT INTO task (...) VALUES (...)",
               params![content, now, due_at])?;
    let task_id = tx.last_insert_rowid();            // ← 取 source_id
    tx.execute("INSERT INTO record (kind, source_id, content, status, created_at)
                VALUES ('task', ?1, ?2, 'pending', ?3)",
               params![task_id, content, now])?;
    tx.commit()?;                                    // ← 提交
    Ok(...)
}

4.4 主窗统一列表（单表查询）

-- 全部记录
SELECT * FROM record ORDER BY created_at DESC LIMIT 50;

-- 隐藏归档
SELECT * FROM record
WHERE kind != 'task' OR source_id NOT IN
  (SELECT id FROM task WHERE archived_at IS NOT NULL)
ORDER BY created_at DESC;

4 状态 TODO 状态机

                     ┌──────────┐
                     │ pending  │  ← 创建
                     └────┬─────┘
                          │ ⏵ 开始计时
                          ▼
                     ┌──────────┐
                     │  active  │  ← Focus 块唯一
                     └────┬─────┘
                          │
              ┌───────────┴───────────┐
              ▼                       ▼
         ┌────────┐              ┌────────┐
         │ paused │              │  done  │  ← Focus 块变体
         └───┬────┘              └───┬────┘
             │ ⏸ 继续                │ ↶ 撤销 / 新任务覆盖归档
             │                       │
             ▼                       ▼
         ┌────────┐              归档
         │ active │
         └────────┘

5 条不变量

  • L1：全局唯一 status='active'（同时只能一个 Focus）
  • L2：paused → active 时 focus_started_at = now，duration_ms 保留
  • L3：active → paused 时 duration_ms += now - focus_started_at，focus_started_at = null
  • L4：active → done 时累加 duration_ms，进入 done 态
  • L5：done → * 不允许终态；可"撤销"回 active
