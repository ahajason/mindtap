浮动窗口形态 v6 — 重新设计待办/计时逻辑

核心洞察：① 展示"待办"和"计时"是同一件事的两面 ② 保存即开始计时 ③ 折叠态只剩 ⏸ 一个核心按钮，整窗可点展开

⚡ 三条核心逻辑（v5 → v6 改变）

① 保存即开始计时：新建待办时，"保存"按钮 = "保存并开始计时"，状态从"未开始"直接进入"进行中"，计时器立即归零启动。无需额外点"开始"。
② 折叠态 = 计时器 = 单一按钮：折叠态只显示 ⏸（暂停/继续）一个核心按钮，其他元素是它的"附属信息"（TODO 标签 + 任务名 + 时间）。整窗可点击/拖动，点击 = 展开。
③ 展开态 = 当前计时 + 新建：上半部分显示当前进行中任务的"完整计时控制"（暂停/继续/完成/切换），下半部分是新建输入（保存即开始计时）。两个时段在一个窗体里。

v6 状态机说明

待办状态：未开始 → 进行中（默认保存即进入） → 完成 · 3 态（保留原 V1.2 的 2 态，新增"进行中"）
计时触发：① 新建待办点保存 → 立即开始计时 ② 切换为进行中 → 启动计时 ③ 暂停 → 计时停但状态仍在进行中 ④ 完成 → 计时归零归档
折叠态展示规则：
  · 有"进行中"待办 → 显示 ⏸ 暂停按钮
  · 无"进行中"待办 → 显示 ⏵ 提示（点击展开去新建或切换）
  · 全部"未开始" → 灰色 idle 状态

3 个逻辑适配方案 → 选 1 / 2 / 3，或多选。
