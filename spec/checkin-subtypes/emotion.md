# 情绪打卡 · 业务规格

> 用户在浮窗以 emoji 单选快速记录每日情绪,可在主窗按时间倒序查看历史与日历视图。

## §1 功能需求（F）

- **F-1** 用户在浮窗快速打卡入口选择"情绪",从预定义 emoji 集合中选取一项提交一条情绪记录
  - R-1.1 emoji 集合为预定义枚举,行业常见 7-9 种(开心/喜欢/平静/思考/焦虑/伤心/生气/犯困/庆祝等)
  - R-1.2 用户单击 emoji 即提交,提交成功后该记录出现在全部记录列表顶部
  - R-1.3 每条情绪记录对应一条打卡记录,且自动出现在统一记录列表中
- **F-2** 用户在主窗历史列表中按时间倒序查看所有情绪记录
  - R-2.1 列表按提交时间倒序排列
  - R-2.2 单条卡片显示 emoji 图标、提交时间、可选备注预览;无备注时省略备注区
- **F-3** 用户可在提交前为本次打卡添加可选的强度评分与短文本备注
  - R-3.1 强度评分与备注均为可选字段,任一为空时 UI 不展示对应区域
- **F-4** 用户可删除任意一条情绪记录
  - R-4.1 删除后该记录从全部记录列表与日历视图中消失
  - R-4.2 删除操作不可恢复
- **F-5** 用户可在主窗查看情绪趋势(月度平均 emoji + 强度均值)
  - R-5.1 趋势按月聚合,无打卡月份不显示
  - R-5.2 强度均值仅在用户填了强度字段时显示
- **F-6** 用户可启用"私密模式"开关,开启后浮窗不显示情绪 note 文本预览(仅显示 emoji)
  - R-6.1 私密模式开关位于主窗设置,默认关闭
  - R-6.2 开启后浮窗 / 通知 / 趋势卡片均隐藏 note 文本,仅显示 emoji 图标

## §2 业务规则（R）

（无）

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供轻量级情绪自我记录,服务用户的日常观察与回顾需求
- **G-2** 与本主题其他子场景共享同一基础记录结构与浮窗入口

### NonGoal

- **NG-1** 本文件不涉及"情绪趋势曲线 / 情绪分析"——留 v2
- **NG-2** 本文件不涉及"AI 情绪识别 / 推断"——留 v2
- **NG-3** 本文件不涉及"情绪社区分享 / 他人可见"——留 v2
- **NG-4** 本文件不涉及"打卡通用数据模型与浮窗入口设计"——本主题所有子场景共享同一基础结构,具体字段约束由各场景自行规定——见 `_cross/foundation.md`
- **NG-5** 本文件不涉及"情绪隐私边界(启动锁 / 浮窗 note 显示策略)"——本主题敏感子场景(情绪/复盘/服药)统一处理——见 `_cross/privacy.md`
- **NG-6** 本文件不涉及"自然日聚合与跨时区归属"——按设备本地时区"自然日"判定——见 `_cross/timezone.md`
- **NG-7** 本文件不涉及"情绪 AI 分析 / 曲线推断"——留 v2
