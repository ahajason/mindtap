# 服药打卡 · 业务规格

> 用户在主窗配置药品与服药时间表,浮窗按当日时间表勾选服用,主窗展示今日依从性。

## §1 功能需求（F）

- **F-1** 用户可在主窗添加一种药品并配置其服药时间表
  - R-1.1 药品配置包含药品名、每次剂量(如"1 片")、剂量单位
  - R-1.2 时间表支持每日定时、每 N 天、每周固定日三种重复模式
  - R-1.3 每个时间表项包含具体时点(如"08:00")
- **F-2** 用户可在主窗查看当日所有药品的时间表与服用状态
  - R-2.1 时间表按时点升序排列
  - R-2.2 每条时点显示药品名、剂量、是否已服用
  - R-2.3 已服用的时点显示服用时间戳
- **F-3** 用户可在浮窗按当日时间表勾选"已服用"提交一条服药记录
  - R-3.1 用户从当日待服列表中勾选某条时点
  - R-3.2 提交后该时点状态变为"已服用",记录服用时间戳
- **F-4** 用户可编辑或删除已添加的药品及其时间表
  - R-4.1 编辑后未来时间表按新配置生成,已发生的服用记录保留
  - R-4.2 删除药品后,所有未来时间表消失,已发生的服用记录保留
- **F-5** 用户可对已过期未服用的时点标记为"跳过"
  - R-5.1 跳过操作记录跳过时间戳,该时点状态变为"已跳过"
  - R-5.2 跳过与已服用互斥,同一时点只能选其一
- **F-6** 用户可标记某条服药时点为"漏服"并查看漏服统计
  - R-6.1 漏服 = 过 schedule_time + 30min 仍未勾选服用或跳过,系统自动标记
  - R-6.2 用户可手动标记任意历史时点为漏服(事后补录)
  - R-6.3 漏服率 = 漏服次数 / 应服次数,按药品分别统计
- **F-7** 用户可设置药品库存量,库存 ≤ 7 天用量时提醒补药
  - R-7.1 库存量由用户手填,系统按 schedule 自动扣减
  - R-7.2 库存提醒走 OS 通知,每日 1 次

## §2 业务规则（R）

（无）

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供日常服药提醒与依从性记录,服务用户的服药规律管理需求
- **G-2** 与本主题其他子场景共享同一基础记录结构

### NonGoal

- **NG-1** 本文件不涉及"药品数据库 / 处方识别"——留 v2
- **NG-2** 本文件不涉及"循环开-停模式(如服药 N 天停 M 天)"——留 v2
- **NG-3** 本文件不涉及"阶梯增减剂量模式"——留 v2
- **NG-4** 本文件不涉及"医院系统对接"——留 v2
- **NG-5** 本文件不涉及"多用户家庭共享"——留 v2
- **NG-6** 本文件不涉及"打卡通用数据模型与浮窗入口设计"——本主题所有子场景共享同一基础结构,具体字段约束由各场景自行规定——见 `_cross/foundation.md`
- **NG-7** 本文件不涉及"药品隐私(浮窗不显示药品名)"——本主题敏感子场景统一处理——见 `_cross/privacy.md`
- **NG-8** 本文件不涉及"依从率统计的图表化"——v1 仅数字,留 v2 图表
- **NG-9** 本文件不涉及"药品相互作用警告"——留 v2(超 MVP 范围,医学风险)
