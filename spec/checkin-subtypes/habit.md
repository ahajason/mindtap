# 习惯打卡 · 业务规格

> 用户在主窗配置习惯目标(频率/提醒时间),浮窗与主窗按日勾选完成,主窗以日历热力图与连续天数展示。

## §1 功能需求（F）

- **F-1** 用户可在主窗添加一个习惯并配置其目标
  - R-1.1 习惯配置包含习惯名、频率(每日/每周 X 次)、提醒时间(可选)
  - R-1.2 用户可对每个习惯设置一个图标
- **F-2** 用户可在浮窗或主窗当日列表中勾选某习惯"今日已完成"
  - R-2.1 已完成的习惯在当日显示勾选状态
  - R-2.2 同一习惯每日仅可勾选一次,二次点击视为取消
- **F-3** 用户可在主窗查看所有习惯的日历热力图与连续天数
  - R-3.1 日历按天聚合,完成日格子着色,未完成日不着色
  - R-3.2 每个习惯显示当前连续天数与历史最佳连续天数
  - R-3.3 连续天数按"连续完成日"计算,中断后从 0 重新累计
- **F-4** 用户可编辑或删除已添加的习惯
  - R-4.1 编辑后未来日历按新配置生成,已完成记录保留
  - R-4.2 删除习惯后,该习惯从所有视图消失,历史记录保留
- **F-5** 用户可对某日某习惯标记"跳过"(非中断)
  - R-5.1 跳过不计入连续天数中断
- **F-6** 用户可设置每周目标次数(如"每周运动 3 次")
  - R-6.1 每周一 0:00 重置本周计数
  - R-6.2 完成次数 ≥ 目标时显示"已达成本周目标"
  - R-6.3 周目标为 0 表示无周目标,仅看每日完成
- **F-7** 用户可标记"休息日",休息日不计入 streak 中断
  - R-7.1 休息日需提前或当日标记
  - R-7.2 休息日不计入也不扣 streak

## §2 业务规则（R）

（无）

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供习惯目标设定与每日打卡追踪,服务用户的习惯养成观察需求
- **G-2** 与本主题其他子场景共享同一基础记录结构

### NonGoal

- **NG-1** 本文件不涉及"习惯关联(做完 A 再做 B)"——留 v2
- **NG-2** 本文件不涉及"成就系统 / 排行榜 / 社交"——留 v2
- **NG-3** 本文件不涉及"惩罚机制 / 放弃情绪分析"——留 v2
- **NG-4** 本文件不涉及"打卡通用数据模型与浮窗入口设计"——本主题所有子场景共享同一基础结构,具体字段约束由各场景自行规定——见 `_cross/foundation.md`
- **NG-5** 本文件不涉及"streak 在 DST 切换日的连续性"——按"自然日"判定,DST 日仍算连续——见 `_cross/timezone.md`
- **NG-6** 本文件不涉及"habit 关联(做完 A 再做 B)"——留 v2
- **NG-7** 本文件不涉及"成就系统 / 排行榜 / 社交"——留 v2
