# 饮食打卡 · 业务规格

> 用户在主窗按早/午/晚/加餐分餐次记录食物条目,主窗以时间线与每日卡路里小计展示。

## §1 功能需求（F）

- **F-1** 用户在主窗"饮食"模块选择某餐次(早/午/晚/加餐)并添加食物条目
  - R-1.1 食物条目包含食物名(必填)、份量描述(可选)、卡路里估值(可选)
  - R-1.2 用户可在同一餐次下添加多条食物条目
- **F-2** 用户可在主窗查看当日饮食时间线
  - R-2.1 时间线按早/午/晚/加餐分段,每段显示该餐次所有食物条目
  - R-2.2 当日时间线底部显示卡路里估值合计(仅当用户填了卡路里)
- **F-3** 用户可编辑或删除某条食物条目
  - R-3.1 编辑后该餐次与当日合计按新内容重算
  - R-3.2 删除操作不可恢复
- **F-4** 用户可删除整餐次的所有条目
  - R-4.1 删除后该餐次从当日时间线消失
- **F-5** 用户可查看本周饮食结构(早午晚条目数 / 卡路里合计)
  - R-5.1 按本周(周一-周日)聚合
  - R-5.2 卡路里合计仅在用户填了卡路里字段时显示
- **F-6** 用户可对某食物条目标记"收藏",收藏条目下次输入时优先联想
  - R-6.1 收藏按用户维度,不分场景共享
  - R-6.2 联想 = 按使用频次倒序前 5

## §2 业务规则（R）

（无）

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供分餐次饮食条目记录,服务用户的饮食结构观察需求
- **G-2** 与本主题其他子场景共享同一基础记录结构

### NonGoal

- **NG-1** 本文件不涉及"食物数据库(卡路里/营养成分自动查询)"——留 v2
- **NG-2** 本文件不涉及"拍照识别食物"——留 v2
- **NG-3** 本文件不涉及"营养分析(蛋白质/脂肪/碳水)"——留 v2
- **NG-4** 本文件不涉及"每日卡路里目标设定与达标追踪"——留 v2
- **NG-5** 本文件不涉及"打卡通用数据模型与浮窗入口设计"——本主题所有子场景共享同一基础结构,具体字段约束由各场景自行规定——见 `_cross/foundation.md`
- **NG-6** 本文件不涉及"食物数据库 / 卡路里库"——v1 食物名自由文本,留 v2
- **NG-7** 本文件不涉及"拍照识别食物"——留 v2
- **NG-8** 本文件不涉及"营养分析(蛋白质 / 脂肪 / 碳水)"——留 v2
