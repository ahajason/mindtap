# 跨子场景通知策略 · 业务规格

> 7 个子场景共享的通知 / 提醒策略,避免各场景独立配置造成疲劳。

## §1 功能需求（F）

- **F-1** 用户可在主窗设置统一"安静时段"(默认 22:00-08:00)
  - R-1.1 安静时段内所有子场景的提醒不弹 OS 通知
  - R-1.2 安静时段内仍可手动打卡
  - R-1.3 安静时段切换静默,不弹"切换"通知
- **F-2** 服药提醒不与同日其他提醒合并,单独走 OS 通知(医学风险)
  - R-2.1 服药漏服 30min 后单独提醒一次
  - R-2.2 不与其他子场景合并
- **F-3** 饮水提醒可"同日合并"为时段提醒
  - R-3.1 默认 6-8 时段(早 / 上午 / 午后 / 下午 / 傍晚 / 晚)
  - R-3.2 用户可关闭合并,改回每杯提醒
- **F-4** 用户可在主窗设置全局通知免打扰(DND)
  - R-4.1 DND 期间所有提醒暂停(手动打卡不受影响)
  - R-4.2 DND 可定时开启(自动勿扰模式)

## §2 业务规则（R）

- **R-0.1** 所有提醒通过 OS 通知发送,需 macOS 通知权限
- **R-0.2** 浮窗不重复弹提醒(只在浮窗可见时显示状态)
- **R-0.3** 同子场景同日同类提醒不重复(去重)
- **R-0.4** 通知文案遵循:emoji + 一句话 + 时间

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供 7 子场景合并后的统一通知配置,避免每个场景独立配置造成用户负担
- **G-2** 平衡"提醒及时"(服药)与"防骚扰"(饮水),提供分层默认

### NonGoal

- **NG-1** 本文件不涉及"AI 智能提醒(根据作息推算最佳提醒时间)"——留 v2
- **NG-2** 本文件不涉及"通知点击直达浮窗 / 主窗"——v1 点击仅打开 app
- **NG-3** 本文件不涉及"7 个子场景各自的专属提醒逻辑"——见 `../water.md` 等
