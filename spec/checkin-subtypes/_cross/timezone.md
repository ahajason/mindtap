# 跨子场景时区与自然日策略 · 业务规格

> 7 个子场景共享的"今天"判定与跨时区处理策略。

## §1 功能需求（F）

- **F-1** 用户打卡时,记录归属当日按设备本地时区的"自然日"判定
  - R-1.1 自然日 = 设备本地时区当日 00:00 至次日 00:00(不含)
  - R-1.2 跨午夜的睡眠 / 复盘按提交时间戳的"自然日"归属
- **F-2** DST 切换日正常记录,渲染层按实际 23h / 25h 显示
  - R-2.1 不做特殊处理,日历按时间戳聚合
- **F-3** 跨时区旅行后,历史记录的日期按各自时间戳的"自然日"显示
  - R-3.1 不回溯重算历史日期
  - R-3.2 用户跨时区后,新打卡按新时区的"自然日"

## §2 业务规则（R）

- **R-0.1** 时间戳精度为毫秒(UNIX epoch ms),时间戳是真相,日期是渲染层计算
- **R-0.2** 习惯 streak 按"自然日"连续计算,DST 日仍算 1 天
- **R-0.3** 饮水 / 饮食 / 复盘的"今日累计"按设备本地"自然日"聚合
- **R-0.4** 服药时间表按时点触发,不绑定"自然日"边界
- **R-0.5** 提醒按设备本地时区触发

## §3 业务边界（G / NG）

### Goal

- **G-1** 桌面单机 + 设备本地时区的简单模型,无需用户配置时区
- **G-2** DST / 跨时区不破坏历史数据完整性

### NonGoal

- **NG-1** 本文件不涉及"用户偏好时区(旅行模式)"——留 v2
- **NG-2** 本文件不涉及"农历 / 其他历法"——v1 仅公历
- **NG-3** 本文件不涉及"7 个子场景各自的专属时间逻辑"——见 `../habit.md` 等
