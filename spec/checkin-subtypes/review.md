# 复盘打卡 · 业务规格

> 用户在主窗按预定义模板填写每日复盘字段,主窗以卡片列表展示历史复盘。

## §1 功能需求（F）

- **F-1** 用户在主窗"复盘"模块选择模板并填写字段提交一条复盘记录
  - R-1.1 系统提供预定义模板,常见模板包括"晨间日记"、"日三件事"、"KPT 复盘法"等
  - R-1.2 每个模板包含 3-7 个固定字段(如"今日成就"、"今日反思"、"明日计划")
  - R-1.3 字段分为短文本(单行)与长文本(多行)两类
- **F-2** 用户可在主窗查看所有复盘记录卡片列表
  - R-2.1 列表按提交时间倒序排列
  - R-2.2 单条卡片显示模板名、提交时间、字段预览
- **F-3** 用户可点击某条复盘记录查看完整字段详情
  - R-3.1 详情页按模板字段顺序展示完整内容
- **F-4** 用户可编辑或删除任意一条复盘记录
  - R-4.1 编辑后字段内容更新
  - R-4.2 删除操作不可恢复
- **F-5** 用户可创建自定义复盘模板(字段 3-7 个,短 / 长文本)
  - R-5.1 模板 = 模板名 + 字段定义(JSON)
  - R-5.2 模板由用户私有,不分场景共享
  - R-5.3 字段标签按当前界面语言显示(中 / 英)
- **F-6** 用户可"复制昨日复盘"作为今日复盘的起点
  - R-6.1 复制 = 字段值复制 + 创建新记录
  - R-6.2 复制后用户可修改字段值再提交

## §2 业务规则（R）

（无）

## §3 业务边界（G / NG）

### Goal

- **G-1** 提供结构化每日复盘记录,服务用户的反思与成长观察需求
- **G-2** 与本主题其他子场景共享同一基础记录结构

### NonGoal

- **NG-1** 本文件不涉及"AI 自动生成复盘内容"——留 v2
- **NG-2** 本文件不涉及"模板分享社区"——留 v2
- **NG-3** 本文件不涉及"跨日聚合视图(如周复盘/月复盘)"——留 v2
- **NG-4** 本文件不涉及"用户自定义模板"——v1 模板为系统预定义
- **NG-5** 本文件不涉及"打卡通用数据模型与浮窗入口设计"——本主题所有子场景共享同一基础结构,具体字段约束由各场景自行规定——见 `_cross/foundation.md`
- **NG-6** 本文件不涉及"AI 自动生成复盘内容"——留 v2
- **NG-7** 本文件不涉及"模板分享社区"——留 v2
- **NG-8** 本文件不涉及"复盘隐私(浮窗不显示模板内容)"——本主题敏感子场景统一处理——见 `_cross/privacy.md`
